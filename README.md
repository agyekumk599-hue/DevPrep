# DevPrep — Exam Practice

A lightweight client-side exam practice app (HTML/CSS/JS) for System Analysis & Design and HTML topics.

## Files

- [index.html](index.html) — main UI
- [script.js](script.js) — application logic, questions, timer
- [style.css](style.css) — styling
- images/ — project images

## Quick Start

1. Open `index.html` in a modern browser (Chrome, Edge, Firefox).
2. Alternatively run a simple local server (recommended) to avoid file:// issues:

```bash
# Python 3
python -m http.server 8000
# then open http://localhost:8000/
```

3. Enter Student Name and ID, choose category, and click **Start Exam**.

## Timer Behavior & Debugging

If the timer shows a static `30:00` and does not count down, use the browser console to inspect logs and errors:

1. Open Developer Tools (F12) → Console.
2. Click **Start Exam** and watch for these log messages:
   - `=== START EXAM CALLED ===` — exam initialization
   - `🔄 Switching view to: exam` — view change
   - `=== START TIMER CALLED ===` — timer setup
   - `Updating timer display to: 30:00 | Timer state: 1800` — timer updates
3. If you see `Timer display element NOT FOUND!` or other errors, the DOM element `timer-display` may be missing or renamed. Verify the element exists in [index.html](index.html).
4. To quickly force-test the timer, open the console and run:

```js
// set remaining seconds to 10 and start the timer (useful for quick checks)
app.state.category = "sad"; // or 'html'
app.state.timer = 10;
app.startTimer();
```

5. To change exam durations, edit the values in [script.js](script.js) inside `startTimer()` (30 min = 1800 seconds; 45 min = 2700 seconds).

## Auto-submit

When time reaches `00:00` the app will automatically submit the exam and navigate to the results view. The app clears the timer interval during submission.

## Troubleshooting Checklist

- Ensure JavaScript is enabled in your browser.
- Do not refresh the page during the exam; that will reset state.
- Check the Console for logs and errors when clicking **Start Exam**.
