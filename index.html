📘 SYSTEM ANALYSIS AND DESIGN (SAD) – REVIEW BOOK

CHAPTER 1: INTRODUCTION TO SAD

System Analysis and Design is a structured process used to develop information systems.

  • System Analysis: focuses on understanding the problem and requirements
  • System Design: focuses on creating the solution

Key idea: Analysis = What is needed; Design = How it will be done

CHAPTER 2: INFORMATION SYSTEMS

An information system is a collection of components that collect, process, store, and output information to support decision-making.

  • Hardware – physical devices
  • Software – programs and applications
  • Data – raw facts turned into information
  • Processes – procedures and operations
  • People – users and IT professionals

CHAPTER 3: SYSTEM ANALYST

  • Understands business problems
  • Designs IT solutions
  • Acts as a bridge between users and developers
  • Key roles: requirements gathering, problem solving, stakeholder communication, design support

CHAPTER 4: SDLC

  • Planning
  • Analysis
  • Design
  • Implementation
  • Support/Maintenance

CHAPTER 5: FEASIBILITY STUDY

  • Technical, Economic, Operational, Schedule

CHAPTER 6: SYSTEM DEVELOPMENT METHODOLOGIES

Waterfall Model - Sequential step-by-step, each phase completes before next. Advantages: Clear structure. Disadvantages: Hard to change requirements.
RAD - Rapid Application Development, fast delivery, tool-based prototyping, user feedback. Advantages: Quick delivery. Disadvantages: Can miss details.
Prototyping - Build a small version first. Advantages: Early feedback. Disadvantages: Risk of poor design.
Agile Development - Iterative, repeated cycles, flexible/adaptive. Advantages: Handles change well.

CHAPTER 7: REQUIREMENTS GATHERING

  • Interviews
  • Questionnaires
  • Observation
  • Document review
  • Sampling
  • Research

👉 Exam Tip: Compare any two methods

CHAPTER 8: SYSTEM MODELING

  • Data Flow Diagram (DFD) - Shows how data moves in a system.
  • Data Dictionary - Stores definitions of data.
  • Process Description Tools:
    • Structured English
    • Decision Tables
    • Decision Trees

CHAPTER 9: OBJECT-ORIENTED APPROACH

  • Object – real-world entity
  • Class – group of objects
  • Attribute – properties
  • Method – actions

CHAPTER 10: UML

Purpose: Used to visually represent system design.

  • Structural Diagrams: Class diagram
  • Behavioral Diagrams: Use case diagram
  • Interaction Diagrams: Sequence diagram

CHAPTER 11: SYSTEM DESIGN

  • User Interface Design
  • Database Design
  • System Architecture

CHAPTER 12: SYSTEM IMPLEMENTATION

  • Coding
  • Testing: Unit testing, Integration testing, System testing
  • Installation
  • Training

CHAPTER 13: SYSTEM SUPPORT & MAINTENANCE

  • Corrective – fix errors
  • Adaptive – adapt to changes
  • Perfective – improve system
  • Preventive – avoid future problems

📘 HTML, CSS & JAVASCRIPT – REVIEW BOOK (DevPrep Edition)

🧩 CHAPTER 1: INTRODUCTION TO WEB DEVELOPMENT

What is a Website?

A website is a collection of web pages accessible via the internet.

Core Technologies:

  • HTML → Structure
  • CSS → Styling
  • JavaScript → Interactivity

🧱 CHAPTER 2: HTML (HYPERTEXT MARKUP LANGUAGE)

Definition: HTML is used to create the structure of web pages.

Basic Structure:

<!DOCTYPE html>
<html>
<head>
  <title>DevPrep</title>
</head>
<body>
  Content goes here
</body>
</html>

Common Tags:

  • <h1> - Heading
  • <p> - Paragraph
  • <a> - Link
  • <img> - Image
  • <div> - Block container
  • <span> - Inline container

Semantic Tags (IMPORTANT ⚠️):

  • <header>
  • <main>
  • <section>
  • <footer>

👉 Improves structure and SEO

🔹 Forms (VERY IMPORTANT)

Used to collect user input:

  • <input type="text">
  • <input type="radio">
  • <input type="checkbox">
  • <button>Submit</button>

🎨 CHAPTER 3: CSS (CASCADING STYLE SHEETS)

Definition: CSS is used to style and design web pages.

CSS Syntax:

selector {
  property: value;
}

Types of CSS: Inline, Internal, External (Best practice ✅)

Colors & Styling:

body {
  background-color: #F8FAFC;
}

📦 CHAPTER 4: THE BOX MODEL (EXAM FAVORITE ⚠️)

Every element consists of:

  • Content
  • Padding
  • Border
  • Margin

👉 Controls spacing and layout

📐 CHAPTER 5: CSS LAYOUT (FLEXBOX)

Flexbox – used for layout and alignment

.container {
  display: flex;
  justify-content: center;
  align-items: center;
}

Key Properties: justify-content, align-items, flex-direction

🖼 CHAPTER 6: IMAGES & LINKS

Images: <img src="image.jpg" alt="description">

👉 alt is important for accessibility ⚠️

Links: <a href="page.html">Go to page</a>

⚙️ CHAPTER 7: JAVASCRIPT (INTERACTIVITY)

Definition: JavaScript makes web pages interactive.

Example:

<button onclick="showMessage()">Click</button>

<script>
function showMessage() {
  alert("Hello DevPrep!");
}
</script>

Key Concepts: Variables, Functions, Events (onclick), Conditions (if/else)

🧠 CHAPTER 8: DOM (DOCUMENT OBJECT MODEL)

Definition: Allows JavaScript to interact with HTML

document.getElementById("demo").innerHTML = "Hello";

🎯 CHAPTER 9: BUILDING A SIMPLE QUIZ SYSTEM

Steps:

  • Display question (HTML)
  • Style interface (CSS)
  • Check answers (JavaScript)

Example Logic:

function checkAnswer() {
  let answer = document.getElementById("q1").value;

  if (answer === "HTML") {
    alert("Correct!");
  } else {
    alert("Wrong!");
  }
}
